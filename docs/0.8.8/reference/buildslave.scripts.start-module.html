startSlave(basedir, quiet, nodaemon)

source code 

Start slave process.

Fork and start twisted application described in basedir buildbot.tac file. Print it's log messages to stdout for a while and try to figure out if start was successful.

If quiet or nodaemon parameters are True, or we are running on a win32 system, will not fork and log will not be printed to stdout.

Parameters:
  • basedir - buildslave's basedir path
  • quiet - don't display startup log messages
  • nodaemon - don't daemonize (stay in foreground)
Returns:
0 if slave was successfully started, 1 if we are not sure that slave started successfully
